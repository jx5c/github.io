Research Interests

My primary research focus on verification of safety and security for autonomous machines. I believe safety-critical systems, such as autonomous cars, should have their properties formally verified. These properties include both functional correctness as well as robustness against adversarial attacks. To do so, software systems for autonomous machines should have formalized models and their properties verified. From the formal model, a large portion of the implementation should be automatically generated, the generation process is formally verified, and the properties verified in the formal model are preserved in the implementation.

Projects

  • Co-Inflow is a coarse-grained dynamic information-flow control mechanism for Java language. By careful design choices and defaults, a programmer typically needs to add very few annotations to a Java program to convert it to a Co-Inflow program with relatively good precision. Additional annotations can improve precision. We achieve this tradeoff between precision and annotation burden by instantiating and specializing recent advances in coarse-grained IFC for a Java-like setting, and by using opaque labeled values: a restriction of labeled values that the Co-Inflow runtime automatically and securely creates and uses.

  • RelSecPhy is a framework for modeling and analyzing sensor attacks on cyber-physical systems, using the formalism of hybrid programs. It formalizes and analyzes two relational properties of a system’s robustness: (1) whether a system’s safety property can be influenced by sensor attacks, and (2) whether a system’s high-integrity state can be affected by sensor attacks. These relational properties is characterized by defining an equivalence relation between a system under attack and the original unattacked system. That is, the system satisfies the robustness properties if executions of the attacked system are appropriately related to executions of the unattacked system.

  • Interpreted formalism is a software analog of the notion of an informal interpretation from classical logic. It targets at improving software dependability by introducing real-world semantics to software systems, and conduct comprehensive type checking based on these semantics.
